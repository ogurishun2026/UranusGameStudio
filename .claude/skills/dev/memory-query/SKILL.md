## 用途

从 MemPalace 记忆宫殿查询项目相关记忆，包括：
- 项目决策和架构设计
- 历史会话记录
- 重要问题解决方案
- 团队成员信息
- 代码模式和最佳实践

## 使用场景

- 会话开始时加载项目上下文
- 回顾某个技术决策的原因
- 查找之前解决类似问题的方案
- 了解项目演进历史

## 执行流程

### Phase 1: 解析查询意图

根据用户输入判断查询类型：

| 关键词模式 | 查询类型 | 优先搜索域 |
|-----------|---------|-----------|
| "架构"、"设计"、"ADR" | 架构决策 | project/architecture |
| "会议"、"讨论"、"决策" | 会话记录 | session/* |
| "bug"、"问题"、"修复" | 问题解决 | code/* |
| "谁"、"团队"、"角色" | 人员信息 | people/* |
| 其他 | 语义搜索 | 全局 |

### Phase 2: 执行查询

```
调用 mempalace_search:
  query: 用户的查询关键词
  limit: 5
  max_distance: 1.5
```

如果查询涉及具体实体（人名、项目名），同时调用：
```
mempalace_kg_query:
  entity: 实体名称
```

### Phase 3: 整合结果

将搜索结果和知识图谱结果整合，按相关性排序：

1. **知识图谱事实** — 确定性高，优先展示
2. **语义搜索结果** — 相似度高的记忆片段
3. **时间线信息** — 如果涉及时间演变

### Phase 4: 输出格式

```markdown
## 记忆查询结果

**查询**: {用户查询}

### 相关事实
- {实体} → {关系} → {对象} (生效于 {日期})

### 相关记忆
1. **{wing}/{room}** (相似度: {score})
   {记忆内容摘要}
   
   来源: {source_file}

### 时间线
{如果有时间相关内容，展示时间线}

---
查询时间: {timestamp}
```

## 示例

### 示例 1：查询架构决策
```
/memory-query 认证系统架构
```

### 示例 2：查询历史问题
```
/memory-query 之前怎么处理数据库连接池问题
```

### 示例 3：查询团队成员
```
/memory-query 后端团队有哪些人
```

## 注意事项

- 如果 MemPalace 未初始化，提示用户运行初始化命令
- 查询结果可能为空，表示没有相关记忆
- 记忆内容是历史记录，可能已过时，需要用户验证
